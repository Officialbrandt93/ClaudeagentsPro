What are these agents?

Understand what you bought before you install it.

Each .md file is a specialist system prompt that transforms Claude into a focused expert — with a specific role, methodology, and output format.

🤖 Without an agent
Claude is a generalist. It answers broadly, has no defined methodology, and delivers inconsistent output quality.
⚡ With an agent
Claude becomes a specialist. It has a defined persona, a step-by-step workflow, and produces structured, professional deliverables.
Think of it this way: You hired 102 specialists — a senior copywriter, a full-stack developer, a brand strategist, a financial analyst... Each agent file "activates" the right one for your task.
Anatomy of an agent file
blog-writer.md
---
name: blog-writer
description: "Use to write long-form SEO articles with
  editorial narrative and brand authority..."
tools: Read, Write, Edit
---

# ↑ Metadata block (frontmatter)
# ↓ System prompt starts here

You are a senior content writer specializing
in long-form SEO editorial content...

## Workflow
1. Analyze search intent
2. Map mandatory topics
3. Write with editorial voice
...

## Deliverables
- Full article (1,500–3,000 words)
- Meta title + meta description
- Internal link suggestions
...
The frontmatter (between the ---) tells Claude when to use the agent automatically. The rest is the agent's personality and methodology.
01

Installation

Choose the method that matches how you use Claude.

The easiest way. Use Projects to give one agent to an entire conversation thread.

  • 1
    Go to claude.ai → click Projects in the left sidebar → New Project
  • 2
    Name your project after the agent (e.g. "SEO Writer" or "Code Reviewer")
  • 3
    Click Project Instructions (top of project page)
  • 4
    Open the .md file of the agent you want, select all text, and paste it into Project Instructions
  • 5
    Save. Every conversation inside this project now uses that agent.
Pro tip: Create a separate Project for each agent you use often. Name them clearly — "📣 Blog Writer", "💻 Code Reviewer", "📊 Financial Analyst".
One agent per project. Don't paste multiple agent files into the same Project Instructions — they'll conflict with each other.

If you use Claude Code (the terminal tool), agents live in a special folder inside your project.

  • 1
    In your project's root directory, create the folder: .claude/agents/
  • 2
    Copy the .md files you want into that folder
  • 3
    Claude Code will detect them automatically on next launch
  • 4
    Invoke manually with /agent agent-name in the chat
Folder structure

Tip: Claude Code will also auto-invoke agents when it decides they're appropriate for the task — that's what the description field in the frontmatter is for.

How to use an agent

The difference between a mediocre and an excellent output is context.

Once an agent is active, you talk to it like a specialist you just hired. Give it context, be specific, and it will deliver structured professional output.

The perfect prompt formula
Prompt structure
[Context]   → What is your business / product / goal?
[Audience]  → Who are you writing/building for?
[Task]      → What exactly do you need?
[Constraints] → Tone, length, format, things to avoid
Blog Writer Agent
You
Write a blog post about marketing.
AI
Here's a generic blog post about marketing...

Marketing is the process of promoting products and services... [500 words of filler content with no clear direction, audience, or angle]
Without context, even the best agent produces generic output. Garbage in, garbage out.
Blog Writer Agent
You
Context: I run a B2B SaaS for HR teams (50–500 employee companies).

Task: Write a 2,000-word SEO article targeting the keyword "employee onboarding software".

Audience: HR Managers who are frustrated with manual onboarding processes.

Tone: Professional but conversational. No jargon. Include 3 real-world examples.
AI
Analyzing search intent... "employee onboarding software" is a commercial investigation query. I'll target HR Managers comparing tools.

Mapping mandatory topics from top-ranking pages...

Here's the full article structure:
H1: The Best Employee Onboarding Software in 2025...
[Structured, SEO-optimized 2,000-word article with examples, meta tags, and internal link suggestions]
What a strong prompt delivers
  • Full article (1,500–3,000 words) with proper H1/H2/H3 structure
  • Optimized meta title and meta description
  • Internal and external link suggestions
  • Editorial checklist for quality and SEO review
  • On-brand tone that matches your audience
Rule of thumb: spend 2 minutes writing a good prompt. It's the difference between 20 minutes of editing and copy-pasting directly to production.
⛓️ Chaining agents for complex projects

The most powerful workflow: use multiple agents in sequence. The output of one becomes the input of the next.

Example: Launch a new product
brand-strategist     → Define positioning + messaging
       ↓
content-planner      → Build content calendar from positioning
       ↓
blog-writer          → Write launch articles
       ↓
ad-copywriter        → Create ad copy from the articles
       ↓
email-sequence-copywriter → Write email nurture sequence
